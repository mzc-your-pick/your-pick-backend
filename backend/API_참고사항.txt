========================================
  API 참고사항 (프론트엔드 개발용)
========================================

1. 요청 헤더
----------------------------------------
모든 POST, PATCH, DELETE 요청 시:
  Content-Type: application/json

GET 요청은 별도 헤더 불필요


2. Swagger UI (API 테스트 도구)
----------------------------------------
브라우저에서 아래 주소 접속하면
모든 API를 직접 테스트해볼 수 있음:

  http://{서버주소}:8080/docs

ReDoc (읽기 전용 문서):
  http://{서버주소}:8080/redoc


3. Base URL
----------------------------------------
모든 API 앞에 붙는 prefix:
  /api/v1

예시:
  GET  http://{서버주소}:8080/api/v1/programs
  POST http://{서버주소}:8080/api/v1/topics

예외 (prefix 없음):
  GET  http://{서버주소}:8080/health
  GET  http://{서버주소}:8080/


4. CORS
----------------------------------------
현재 모든 도메인에서 접근 허용 (allow_origins=*)
프론트에서 별도 CORS 설정 필요 없음


5. Rate Limit (요청 제한)
----------------------------------------
- 투표 (POST /topics/{id}/votes) : 분당 10회
- 결과 조회 (GET /topics/{id}/results) : 분당 60회
- 제한 초과 시 429 에러 반환


6. 투표 흐름 요약
----------------------------------------
1) 프로그램 목록 조회  GET /programs
2) 주제 목록 조회      GET /topics?program_id=1
3) 투표하기            POST /topics/{topic_id}/votes
4) 결과 확인           GET /topics/{topic_id}/results
5) 댓글 작성           POST /votes/{vote_id}/comments
6) 댓글 조회           GET /topics/{topic_id}/comments


7. vote_type 설명
----------------------------------------
1 = 합불 (합격/불합격)
2 = 1대1 (두 참가자 중 선택)
3 = 다인원 (여러 참가자 중 선택)


8. actual_result 설명
----------------------------------------
- 방송에서 나온 실제 결과 (관리자가 입력)
- null이면 아직 방송 결과 미입력
- 결과 조회 시 match 필드:
  true  = 대중 투표 1위와 방송 결과 일치
  false = 불일치
  null  = 방송 결과 미입력
